# 全链路 SOP 总控入口

本文件是 7 阶段工作流的统一调用入口。目标不是一次性生成全文，而是按阶段稳定推进，并让每个阶段都有明确输入、输出和交接物。

## 使用方式

1. 按阶段把资料填入项目目录
2. 每到一个阶段，就复制对应阶段 prompt 给模型
3. 阶段完成后，把结果写回项目对应目录
4. 每个阶段末尾都必须保留人工干预位
5. 阶段 7 结束后，把有效结论回写 L1
6. 每日审阅交付统一导出到桌面日期目录

## 阶段顺序

1. `01_内容采集_prompt.md`
2. `02_内容聚合及选题分析_prompt.md`
3. `05_初稿生成_prompt.md`
4. `04_素材收集_prompt.md`
5. `06_改写_prompt.md`
6. `07_渠道分发_prompt.md`
7. `08_分析复盘_prompt.md`

当前阶段语义定义见：`引擎/03_全链路SOP工作流/STAGE_INTERFACES.md`

## 总控规则

- 任何阶段都不能杜撰事实、数据、引用
- 缺失信息必须明确写“素材未提供/证据不足”
- 所有判断都尽量区分【事实】【推断】【建议】
- 阶段 1-4 偏事实整理与素材补齐；阶段 5-6 才进入表达层与分发层
- 同一项目必须保持一个主判断，不允许阶段间漂移
- 第 3 阶段只生成标准初稿，不做账号风格注入，不做平台口吻适配
- 第 3 阶段正文不得引用任何内部流程统计；如需写入文章，必须回到外部原始来源或重新检索公开数据
- 第 3 阶段默认控制在 `3-4` 个一级标题，优先通过二级标题组织层次
- 第 3 阶段必须主动补充外部数据、表格和权威引用，不做机械扩写
- 第 4 阶段只补证据与素材，并直接回填到编辑确认后的终稿文档，不允许借机改主判断
- 第 5 阶段必须基于回填后终稿做多版本改写，不得回退到旧初稿
- 每个阶段末尾都必须输出一个“人工干预位”，明确哪些内容允许人工增删改
- 每日审阅交付统一导出到：`/Users/lichengyin/Desktop/自媒体创作临时交付/<日期>/`

  - `python3 /Volumes/PSSD/Projects/公众号文章/scripts/feishu_stage_sync.py <run_id>`
  - `python3 /Volumes/PSSD/Projects/公众号文章/scripts/feishu_stage_sync.py --latest`
- 断点续跑：
  - `python3 /Volumes/PSSD/Projects/公众号文章/scripts/feishu_stage_sync.py --resume-only <run_id>`

  - 飞书同步中断后，优先使用 `--resume-only`
  - 只有确认旧进度不可恢复时，才使用 `--fresh`
  - `--fresh` 与 `--resume-only` 不能同时使用

## 总控提示词

```text
你现在不是一次性写完整篇文章，而是扮演“自媒体创作 SOP 执行器”。

我会按 7 个阶段逐步给你输入材料。你的任务是：
1. 只处理当前阶段，不越级提前写后面的内容。
2. 先识别当前阶段目标，再识别输入是否完整。
3. 输出必须包含：
   - 本阶段结论
   - 缺失项
   - 下一阶段交接物
   - 建议保存到项目目录的文件内容
   - 人工干预位
4. 不允许编造数据、来源、截图、采访、观点引用。
5. 如材料不足，先帮我列出缺口和补料建议，而不是硬写。

当前项目使用的总方法：
- 风格约束遵循：`引擎/01_调性分析引擎/STYLE_KNOWLEDGE_BASE.md`
- 风格/多版本改写遵循：`引擎/02_三版写作引擎/L2_总控提示词_三版.md`
- 流程遵循：`引擎/03_全链路SOP工作流/README.md`

当前阶段：<填写阶段名>
当前项目：<填写项目名>
当前输入：<粘贴本阶段素材>
```

## 阶段交接最小集合

| 阶段 | 必须交给下游的东西 |
| --- | --- |
| 01 Intake | 原始链接清单、来源摘要、采集结论 |
| 02 Brief | 母题聚类、主选题、核心判断、大纲、来源包 |
| 03 Draft | 分题标准初稿、证据清单、待补证据项 |
| 04 Material | 补充数据、图片、引用、终稿回填说明 |
| 05 Rewrite | 分题 4 版改写、字数统计、改写汇总包 |
| 06 发布 | 各渠道成品包、发布时间计划、发布主链接 |
| 07 复盘 | 结果结论、失效点、回写 L1 的条目 |
