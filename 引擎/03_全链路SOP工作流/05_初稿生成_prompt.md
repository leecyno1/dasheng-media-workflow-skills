# 05 初稿生成 Prompt

## 阶段目标

基于已确认的 `Brief`、外部来源与可复核数据，生成一篇标准初稿，作为后续编辑、补素材、改写与分发的唯一事实底稿。

**核心工作流**：Draft 环节是人工 + AI 迭代循环，直到编辑 approve 或确认”中版”。

## 适用输入

- 已确认选题
- Brief / 大纲
- 外部可复核来源
- 已收集素材
- 编辑补充要求（可选）
- **编辑投喂的范文**（如有）
- **编辑的修改意见**（如有）

## AI 生成触发条件

编辑通过以下方式触发 AI 生成：
1. 给出明确Brief（含题目、大纲、写作要求）
2. 投喂范文（1篇或多篇），要求AI结合范文生成

## 迭代循环流程

```
编辑修改内容 → 使用@标注批改意见 → AI修改相应部分 → AI删除所有@锚点 → 编辑再审
    ↑                                                                              ↓
    ←←←←←←←←←←←←← 直到编辑发送”approve”/”通过”/”终版”←←←←←←←←←←←←←←
```

**批改意见标注格式**：
```text
@这段话需要改成...@
@这个论点证据不足，需要补充...@
@第三段的逻辑有问题，重写...@
```

**AI修改完成后**：必须删除文件中所有 `@` 和 `@...@` 锚点标记。

**循环终止条件**（编辑发送以下任一指令，本轮结束）：
- `”approve”` 或 `”通过”`
- 确认”中版”或”终版”（如：`稳定币那篇是终版`）
- 明确指示”进入下一环节”

**批改意见标注规范**：
- 编辑使用 `@批改意见示例@` 标注需要修改的内容
- AI完成修改后，**必须删除所有 `@` 锚点标记**

## 输出要求

1. 标准初稿正文
2. 证据清单
3. 待补证据项
4. 风险审查
5. 人工干预位
6. 飞书协作交接

## Prompt

```text
你现在负责自媒体创作 SOP 的 Draft 环节。

请基于已确认的选题、Brief、大纲和外部可复核来源，生成一篇”标准初稿”。

你必须遵守：
1. 这是标准稿，不是平台稿；不得提前注入公众号、小红书、短视频等渠道腔。
2. 这是事实底稿，不是账号风格稿；不得提前注入鲁迅、Lemon、表舅等 DNA。
3. 正文事实、数据与判断，必须来自外部可复核来源；不得把 intake / brief / cluster 的内部统计写进正文。
4. 正文目标字数建议 4000-8000 字，但**不强制限制字数**，以内容完整、论证充分为准。
5. 结构上默认只采用两种骨架：`开篇 + 三段论 + 结尾`，或 `开篇 + 4 个一级章节 + 结尾`。
6. 一级标题不得超过 `4` 个；允许在一级标题下使用二级标题增强层次。
7. `brief` 给出的五六条大纲仅作参考线索；你必须重新提炼作者主旨，决定哪些写、哪些不写。
8. 先给判断，再拆证据，再收束到框架；大段推断必须压缩，证据不足处只能降级表达。
9. 你必须主动补充外部数据与表格，优先使用 `akshare`、`tushare`、官方统计、主流财经媒体与国际媒体，不得被动只复述上游大纲。
10. 在合适位置可以插入 markdown 表格，帮助编辑快速审阅关键变量、时间线或对比关系。
11. 不得生成爆款标题池、平台 CTA、口播句式、情绪化煽动句。
12. 每个选题必须形成一篇独立飞书文档，供编辑团队并行审核；不得把多个题目合并成一篇长文档。
13. 编辑团队对初稿文档修改后，该文档视为后续 `material` 阶段的终稿底板。
14. **如编辑提供范文**：分析范文的结构、文风、论证逻辑，结合编辑的Brief要求生成新稿。
15. **如编辑给出修改意见**：严格按意见修改指定部分，不扩大修改范围。

请按以下结构输出：

一、成稿说明
- 选题：
- 主判断：
- 目标读者：
- 字数目标：
- 采用结构：`三段论` / `四段式`
- 本稿核心证据来自：

二、标准初稿正文
- 输出完整标准稿正文
- 正文默认结构：
  - 开篇
  - 一级章节 1（可含二级标题）
  - 一级章节 2（可含二级标题）
  - 一级章节 3（可含二级标题）
  - 一级章节 4（可选）
  - 结尾
- 若使用数据对比、时间线或变量拆解，优先插入 markdown 表格
- 在需要配图/链接的位置使用锚点标注：
  - `{{image: 描述内容}}` — 配图占位
  - `{{link: URL|显示文字}}` — 链接占位
  - `{{ref: 来源名称}}` — 参考文献标注

三、证据清单
- 关键证据 1：来源名 + 链接/接口
- 关键证据 2：来源名 + 链接/接口
- 关键证据 3：来源名 + 链接/接口
- 如有主动补充数据，写明接口与字段口径

四、待补证据项
- 仍需补的数据：
- 仍需补的案例：
- 仍需补的图表：

五、风险审查
- 当前最弱的一段论证：
- 哪些判断仍需降级表达：
- 哪些地方不能继续外推：

六、人工干预位
- 哪些段落允许删减：
- 哪些小标题允许调整：
- 人工改动后必须回查的事实点：

七、飞书协作交接
- 飞书初稿标题：
- 飞书初稿报告标题：
- 飞书群消息摘要：
- 建议编辑优先审的章节：
- 飞书文档拆分规则：`一题一文档`

八、相关素材链接（来源：intake 环节）
- 相关文章链接（标题 + URL）
- 相关视频链接（标题 + URL）
- 如无相关素材，写明"本篇无直接相关 intake 记录"

当前输入材料如下：
<粘贴已确认选题、Brief、大纲、外部来源与用户补充要求>
```

## 完成检查

- 是否只产出标准稿，而非风格稿/平台稿
- 是否达到合理字数区间（建议4000+）
- 是否把一级标题控制在 `3-4` 个以内
- 是否用二级标题而不是继续拆更多一级标题
- 是否全部基于外部可复核来源
- 是否主动补充了必要数据 / 表格 / 时间线
- 是否为后续编辑终稿与补素材预留了明确接口
- 是否在需要配图/链接处标注了锚点
