# 08 复盘 Prompt

## 阶段目标

基于发布后的真实表现，判断这篇内容起量或失效的原因，并把结论回写到风格知识库和选题模型。

## 适用输入

- 已发布内容
- 标题版本
- 阅读/播放/点赞/评论/转发等数据
- 用户反馈
- 渠道差异表现

## 输出要求

1. 表现总评
2. 起量 / 不起量原因
3. 选题和写法复盘
4. 模型修正建议
5. 回写 L1 条目
6. 人工干预位

## Prompt

```text
你现在负责自媒体创作 SOP 的第 8 阶段：复盘。

请根据发布结果，复盘这篇内容为什么有效或无效，并输出后续可执行的修正建议。

你必须遵守：
1. 复盘要区分：选题、结构、标题、证据、渠道、发布时间。
2. 不要只说“数据一般/效果不错”，必须解释原因。
3. 必须给出哪些做法要继续、哪些要停止、哪些要继续测试。
4. 最终要产出可以回写到 L1 的新规则或修正规则。

请按以下结构输出：

一、表现总评
- 核心数据概览
- 是否达到预期

二、原因拆解
- 选题层面：
- 标题层面：
- 开头层面：
- 证据层面：
- 渠道层面：

三、有效动作
- 哪些动作明显有用

四、失效动作
- 哪些动作没有带来效果

五、后续动作
- 继续做：
- 停止做：
- 继续测试：

六、回写 L1 风格知识库
- 新增规则：
- 修正规则：
- 待观察点：

七、人工干预位
- 建议人工确认的数据口径：
- 建议人工增补的复盘备注：
- 确认后需回写到哪些文档：

当前输入材料如下：
<粘贴发布结果和数据>
```

## 完成检查

- 是否解释了结果而非只复述数据
- 是否给出继续/停止/测试三类结论
- 是否产出可回写 L1 的条目
- 是否为人工补写复盘留出明确位置
