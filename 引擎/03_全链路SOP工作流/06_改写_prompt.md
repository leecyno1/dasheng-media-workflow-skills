# 06 改写 Prompt

## 阶段目标

把回填后终稿，按 `DNA × Channel × Emotion` 改写成多个可发布版本。

## 适用输入

- 回填后终稿
- 可选 Brief
- 可选 Material Pack
- 目标 DNA
- 目标 Channel
- 目标 Emotion
- 可选用户附加要求

## 输出要求

1. 单版本改写正文
2. 标题池
3. 改写说明
4. `meta.json`
5. `rewrite_bundle.md`
6. `rewrite_manifest.json`
7. 飞书协作交接

## Prompt

```text
你现在负责自媒体创作 SOP 的 rewrite 环节。

你的任务不是简单润色，而是把回填后终稿改写成一个“适配指定文风 DNA、指定发布渠道、指定情绪档位”的可发布版本。

你必须严格遵守以下规则：

【硬规则】
1. 不得改变已确认事实、数据方向、核心判断和风险边界。
2. 不得引入新证据；本阶段不负责新增素材回填，只在既有终稿事实边界内改写。
3. 先按渠道重排结构，再按 DNA 改写语言，最后再应用情绪档位。
4. 改写时要同时生成标题池与改写说明，不可只给正文。
5. 多个选题默认多个 Agent 并行，每个 Agent 只负责一个选题，不允许混稿。
6. 每个选题默认生成 4 个版本，并汇总到 `rewrite_bundle.md`。
7. 本阶段结束时，必须按“每题一条消息”把字数统计和飞书文档链接发送到群。

【三维参数】
- DNA：<填写 dna_id，例如 luxun / lemon / biaoshu / standard-neutral>
- Channel：<填写渠道，例如 wechat_article / wechat_video / xiaohongshu_note / xiaohongshu_video / douyin_video / bilibili_video / video_account_video / weibo_note / weibo_video / x_note / x_video>
- Emotion：<填写情绪档位，例如 joy / hot / normal / cool / angry>

【默认方法】
- 优先服从渠道约束
- 在渠道约束内执行 DNA
- Emotion 只做强度微调
- 如果 DNA 与渠道冲突，以渠道适配为先

【字数与结构硬约束】
- 若 Channel = `wechat_article`，正文目标字数锁定 `4000-8000` 字，推荐落在 `4500-6500` 字。
- 若上游终稿本身已在 `4000-8000` 字之间，改写不得无故压缩到 `4000` 字以下。
- 若改写后可能低于下限，必须通过补充证据展开、反例辨析、影响拆解、风险提示和结尾收束把正文补足。
- `wechat_article` 必须继承上游终稿的一级结构：一级标题数量不得高于上游终稿（最多允许 `+1`），不允许强制改造成固定三段论/四段论。
- 补字只能在既有章节内扩写二级段落，不得通过新增多个一级标题凑字数。
- 若 Channel 属于任一 `*_video`，默认按 `6` 分钟视频脚本生成，正文目标字数 `1800-2800` 字；`bilibili_video` 默认 `2200-3400` 字。
- 视频稿按每分钟 `300` 字估算，不得再输出 `120-300` 字级别的短文案。
- 视频稿必须写成可录制的完整口播脚本，至少包含 `6` 个实质口播段。

【输出结构】
请按以下结构输出：
```

## 完成检查

- 是否严格维持事实边界
- 是否明显体现指定 DNA
- 是否明显适配指定 Channel
- 是否正确控制了 Emotion 档位
- 是否通过结构继承校验（一级标题数量不超过上游终稿上限）
- 是否满足目标字数区间
- 是否给出了标题池与改写说明
