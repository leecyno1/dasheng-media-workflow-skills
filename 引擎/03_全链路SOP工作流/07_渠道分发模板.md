# 渠道分发模板

本模板用于阶段 7，把同一篇终稿拆成不同平台的可发布包。

## 总原则

- 核心判断一致，表达节奏因平台而变
- 不同渠道分别优化标题、开头、CTA、素材密度
- 每个渠道都要有独立的发布清单，不能只复制正文

## 公众号分发包

- 标题 5 个：16-28 字，至少 1 个具象词或数字
- 导语 80-120 字：先结论，再给读者收益
- 正文：保留完整结构与图表占位
- 封面建议：主标题 + 1 个副标题
- 文末 CTA：评论问题 + 关注下一篇预告

建议字段：

```text
标题：
导语：
封面文案：
首图链接：
正文版本：
文末互动问题：
```

## 小红书图文分发包

- 标题 2 套：一套偏冲突，一套偏方法
- 封面文案：12-18 字，适合大字图
- 页卡结构：封面 / 观点 / 证据 / 方法 / 结论 / CTA
- 标签：5-8 个，不堆砌
- 正文：更短句、更口语、更强调“我替你总结”

建议字段：

```text
标题：
封面文案：
页卡结构：
图片来源：
标签：
评论区引导：
```

## 视频号 / 抖音口播包

- 前 3 秒 Hook：冲突句或反常识结论
- 正文：60-120 秒口播稿，按字幕节奏拆句
- 镜头建议：口播 / 数据截图 / 图表切换 / 总结卡
- 结尾 CTA：关注、评论、下期预告

建议字段：

```text
口播标题：
3秒 Hook：
分镜脚本：
字幕关键词：
结尾 CTA：
```

## 社群 / 朋友圈转发包

- 40-80 字摘要：只讲一个判断
- 带 1 个问题，方便引出讨论
- 附原文链接或主图链接

建议字段：

```text
转发摘要：
引导问题：
链接：
配图：
```

## 发布前检查

- 标题是否针对渠道单独优化
- 首屏信息是否在 3 秒内能看懂
- 图表和图片链接是否可点击
- 风险提示是否保留
- 是否准备好评论区第一条置顶内容
