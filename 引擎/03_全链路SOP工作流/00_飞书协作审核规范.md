# 飞书协作审核规范

更新时间：`2026-04-01`

## 目标

把 `intake -> brief -> draft -> material -> rewrite` 五个阶段统一接入飞书共享协作层，让编辑团队可以在每一阶段直接看到文档、介入修改、审核取舍。

## 强制规则

1. 每个阶段完成时，必须同时形成：
   - 本地正式交付文档
   - 飞书共享文档
   - 飞书群消息
2. 飞书共享文档必须放进当日日期文件夹。
3. `material` 阶段必须把素材目录上传到飞书日期文件夹。
4. `draft` 阶段正文必须按题拆成独立飞书文档，供编辑团队并行审核。
5. 编辑团队在 `draft` 飞书文档上的修改，视为后续 `material` 阶段的终稿工作底板。
6. `material` 阶段必须直接编辑飞书终稿文档，把图、表、数据图回填到对应锚点或对应章节位置。
7. `rewrite` 阶段必须基于回填后终稿执行，不得回退到旧 draft。
8. 未完成飞书协作同步的阶段，不视为“可进入 publish”。

## 日期文件夹规范

- 命名：`YYYY-MM-DD`
- 位置：飞书团队共享空间下的自媒体创作目录
- 最低结构：
  - `01_Intake`
  - `02_Brief`
  - `03_Draft`
  - `04_Material`
  - `05_Rewrite`

## 飞书群消息规范

每个阶段群消息至少包含：

- 阶段名
- 当前选题 / 当前 run_id
- 关键结论摘要（2-5 条）
- 飞书文档链接
- 如有素材文件夹，再附文件夹链接
- 需要编辑人工介入的点

## 失败恢复规范

- 飞书同步中断、超时、意外退出时，默认先执行：
  - `python3 /Volumes/PSSD/Projects/公众号文章/scripts/feishu_stage_sync.py --resume-only <run_id>`
- `--resume-only` 会复用已有进度，跳过已完成 action，只补未完成部分。
- 只有在以下情况才允许执行：
  - `python3 /Volumes/PSSD/Projects/公众号文章/scripts/feishu_stage_sync.py --fresh <run_id>`
- 适用条件：
  - 旧进度文件损坏
  - 文档/文件夹映射明显错乱
  - 业务上明确要求整套重建
- 禁止同时使用 `--fresh` 和 `--resume-only`。
- 进度文件固定位置：
  - `skills/dasheng-daily-shared/runtime-data/runs/<run_id>/bridge/live-execution-progress.json`
- 编辑团队操作原则：
  - 先续跑，再重跑
  - 非必要不重建已有飞书文档，避免覆盖人工修改痕迹

## Material 回填规范

终稿文档中的锚点统一作为 `material` 的回填坐标；若终稿未保留锚点，则以章节位置为准。

### 表格策略（新增）

1. `draft` / `rewrite` 正文阶段禁止写入大表（>12 行原始数据表），仅保留结论性表述与“数据来源说明”。
2. 数据表统一在 `material` 阶段由 CSV 回填，写入飞书原生 table block。
3. 同一锚点下的 CSV 仅保留前 `12` 行可视化预览；完整 CSV 留在素材包目录供编辑查看。
4. 严禁把表格降级为“按行拼接文本（key:value）”写入飞书正文。

### 幂等回填策略（新增）

1. 每次回填前，先清空 `素材回填区` 标题下的旧内容，再执行本轮回填。
2. 每次写入视频链接前，先清空 `视频素材链接` 标题下的旧内容，再写入一次。
3. 若重复运行同一批次，只允许覆盖，不允许追加同批重复素材。

回填优先级：

1. 图表
2. 数据表格
3. 图片
4. 信息图
5. 视频素材链接注释

回填后必须在 `05_Material_报告` 中记录：

- 已回填锚点
- 未回填锚点
- 待人工替换项


## 并行创作规范（2026-04-01）

### 核心原则
- `intake → brief` 保持串行（全局信息聚合必须先完成）
- `draft → material → rewrite` 按选题切分为独立 Agent，并行执行
- 每选题独立交付飞书文档，编辑团队可并行审核
- 最终 `publish` 阶段再汇总所有选题

### Agent 分工模板
- **并行 Agent 1**：选题 1 的 `draft + material + rewrite`，独立飞书文档
- **并行 Agent 2**：选题 2 的 `draft + material + rewrite`，独立飞书文档
- **并行 Agent N**：选题 N 同上
- **主 Agent**：协调派发、汇总报告、触发 `publish`

### 飞书协作约束（并行场景）
- 每个选题的 `draft` 初稿写入独立飞书文档（如 `03_标准初稿/题目1`、`03_标准初稿/题目2`）
- 每个选题的 `material` 素材包独立文件夹，按题目命名
- 每个选题的 `rewrite` 改写稿写入独立飞书文档（如 `05_Rewrite/题目1`、`05_Rewrite/题目2`）
- 飞书群消息按选题分条发送，标题注明「选题N：XXX」

### 启动方式
- 串行模式（单题调试）：`python3 scripts/material_execute_pack.py --draft-manifest <draft_manifest.json> --topic-dir <topic>`
- 并行模式（多题，推荐）：`python3 scripts/material_parallel_launcher.py --draft-manifest <draft_manifest.json>`
- 并行复跑模式（已有 canonical material 清单时）：`python3 scripts/material_parallel_launcher.py --material-manifest <material_manifest.json> --topics <topic-6 topic-10>`
- 并行 run 入口：`python3 scripts/material_parallel_launcher.py --run-id <run_id> --topics <topic-6 topic-10>`

### 素材执行门槛（并行场景复用）
- 图表质量门控：CV≥0.03 或 |slope|≥0.005 或 R²≥0.25；无趋势改为逻辑关系表
- 图片分辨率门槛：短边 ≥1024px；高频人物图优先；含新闻网页截图通道；搞笑漫画人物分支
- 视频质量规则：不设固定分辨率门槛；过滤口播自媒体出镜；防截图拼视频三重校验（来源/时长/帧变化）
- Layer5 互动页：每个选题独立输出 `layer5_inputs.json` + `logic_relations.json`，接 worldmonitor 端口 8787

## 发布前门槛

只有同时满足以下条件，才允许进入 `publish`：

- 前五阶段飞书共享文档已创建
- 对应群消息已发送
- Material 素材文件夹已上传
- 终稿首轮回填已完成
- Rewrite 4 版已生成并完成字数校验
- 待人工确认项已显式标出
