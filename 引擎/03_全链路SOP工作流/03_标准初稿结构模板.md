# 03 标准初稿结构模板

## 目标

把 `brief` 交付的题目、结构骨架和来源包，转成一篇可供编辑直接修改的标准长文底稿。

这不是平台稿，也不是风格稿，而是后续所有改写、补素材、分发的唯一事实母稿。

## 默认结构

默认只采用以下两种主结构之一：

1. `开篇 + 三段论 + 结尾`
2. `开篇 + 4 个一级章节 + 结尾`

强制约束：

- 一级标题默认 **不超过 4 个**，绝不写成 6-8 个碎片章节。
- 每个一级标题下允许设置多个二级标题，用于承接子问题、数据表、案例或反方辨析。
- `brief` 给出的 5-6 条大纲只作为线索，不要求机械写全。
- 写作时必须先重新提炼：
  - 这篇文章到底要证明什么
  - 读者真正关心的冲突是什么
  - 哪三到四个论证块最能支撑主旨

## 建议骨架

### 1. 开篇

- 只做一件事：明确冲突和主判断。
- 不要从流程、背景、定义写起。
- 开篇 3-6 段内完成：
  - 当前发生了什么
  - 市场/舆论常见误读是什么
  - 作者真正要讨论的主线是什么

### 2. 第一章：主矛盾

- 解释文章的第一层判断。
- 可挂 2-3 个二级标题：
  - 事实证据
  - 时间线 / 历史参照
  - 关键变量

### 3. 第二章：证据展开

- 把最关键的数据、案例、政策、财务口径摆出来。
- 允许插入 markdown 表格。
- 金融/宏观题尽量主动补：
  - `akshare`
  - `tushare`
  - 官方统计
  - 主流财经媒体

### 4. 第三章：反方与边界

- 处理最容易被误读的地方。
- 说明：
  - 什么不能外推
  - 哪些只是阶段现象
  - 哪些证据仍然不足

### 5. 第四章（可选）：影响与策略

- 只在确有必要时保留。
- 适合承接：
  - 产业链影响
  - 资产价格影响
  - 读者行动框架

### 6. 结尾

- 回到最初判断。
- 给出收束，而不是口号。
- 结尾默认回答：
  - 这篇文章读完后，读者应如何重新理解这个题目

## 数据与表格

标准初稿允许主动插入 markdown 表格，尤其适用于：

- 时间线
- 指标对比
- 公司/地区/国家横向比较
- 情景分析

示例：

| 变量 | 当前变化 | 对主判断的意义 |
| --- | --- | --- |
| 通胀预期 | 回升 / 反复 | 影响利率重估 |
| 油价 | 上行 / 波动 | 影响再通胀叙事 |
| 利率 | 粘性更强 | 压缩风险资产估值 |

## 飞书交接

- 每个选题形成 **一篇独立飞书文档**，不要把 3 个题合成一个超长文档。
- 初稿报告可以汇总，但正文必须按题拆开，便于编辑并行审核。
- 编辑改完后，该文档即视为后续 `material` 阶段的终稿工作底板。

## 禁止项

- 不得照抄 `brief` 大纲。
- 不得把内部流程统计写入正文。
- 不得把一级标题拆得过碎。
- 不得把没有证据的数据判断写成结论。
- 不得提前做平台腔、口播腔、标题党包装。
