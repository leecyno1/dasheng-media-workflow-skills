# STYLE KNOWLEDGE BASE（自媒体风格知识库）

> 用途：沉淀「已验证有效」的写作模式。由 L1 引擎持续迭代。

## 1) 标题模式（已验证）

- `情绪词 + 数字 + 反常识结论`
  - 例：`今天还有2.4亿...我有点看不懂了`
- `风险提示 + 行动导向`
  - 例：`一定要高度重视背后的影响`
- 标题长度建议：16-28 字，至少 1 个具象数字。

## 2) 开头模式（已验证）

- 前 120-180 字必须同时给出：
  - 事件冲突
  - 作者结论
  - 读者收益（为什么要继续看）
- 开场避免背景铺垫过长，默认“先结论后背景”。

## 3) 正文结构（已验证）

- 主结构：`01 发生了什么 -> 02 为什么 -> 03 怎么做`
- 每节 2-5 个子点，单点先结论再证据。
- 关键段落保持短句，避免连续大段说明文。

## 4) 证据规则（已验证）

- 每个核心观点至少 1 条证据：
  - 数据（同比/环比/分位）
  - 公告或政策原文
  - 跨市场映射（A/H/美、商品与权益）
- 观点类型标注：`[事实]` / `[推断]` / `[建议]`。

## 5) 语言与语气（已验证）

- 口语化表达提升完读率，但必须保留逻辑骨架。
- 情绪强度可高，但结论要可复核。
- 高频句式：`不是X，而是Y`、`先看结论`、`这里给到操作框架`。

## 6) 排版节奏（已验证）

- 每 300-500 字插入一个信息锚点（结论句/数据句/图）。
- 段落长度控制在 1-3 句。
- 图表占位写法统一：`[图1：xxx]`。

## 7) 转化动作（已验证）

- 文末固定三件事：
  1. 一句话结论
  2. 合规风险提示
  3. 互动问题 + CTA（评论/在看/关注下一篇）

## 8) 版本策略（与 L2 联动）

- A版：平衡型（可读性+可信度）
- B版：流量型（标题冲突+快节奏）
- C版：专业型（机制链条+失效条件）

## 9) 待迭代观察点

- 哪类标题更稳定带来在看率（而非只提高点击）
- 情绪强度与完读率是否存在拐点
- 图表数量与转发率的最优区间

## 10) 阶段8回写（2026-03-22_200511）

### 新增规则（先执行，按数据复核）

- 无任务级前后对照数据时，正文与短内容都不写“提效X%”等量化承诺。
- 跨渠道分发时，核心判断必须保持同一句结论，只调整篇幅与节奏。
- 评论区优先使用“二选一痛点问题”收集可执行反馈。

### 修正规则

- 标题优先级修正为：`冲突句 + 方法名 + 使用场景`，再叠加情绪词与数字。
- 渠道改写时保留“风险边界”信息，不因追求点击而删除。

### 待观察点（本轮新增）

- 冲突型标题是否提升点击但拉低完读率。
- 保留风险提示后，对转发率和评论质量的净影响。
- 60-90 秒口播长度对完播率的影响区间。
