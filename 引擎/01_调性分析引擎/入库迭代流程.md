# 入库迭代流程（新增项目必走）

## Step 1. 新建项目

```bash
python3 scripts/new_content_project.py --name "2026-02-xx_选题名"
```

## Step 2. 素材抓取与落盘

- 微信链接抓取后会得到 `index.json`
- 把 `index.json` 路径记录到项目 `inputs/materials_manifest.md`

## Step 3. 入库分析（L1）

```bash
python3 scripts/style_ingest.py \
  --index data/wechat_scrapes/<run_id>/index.json \
  --project "2026-02-xx_选题名" \
  --tag "wechat" \
  --author "表舅是养基大户"
```

执行后会生成：
- `引擎/01_调性分析引擎/analysis_runs/<timestamp>_<project>.md`
- `引擎/01_调性分析引擎/style_registry.jsonl`

## Step 4. 知识库更新

- 把新增、稳定、失效模式回写 `STYLE_KNOWLEDGE_BASE.md`
- 原则：只保留可复用规则，不记录一次性噪音

## Step 5. 三版写作（L2）

- 使用 `引擎/02_三版写作引擎/L2_总控提示词_三版.md`
- 一次输出 A/B/C 三稿，进入项目 `outputs/`

## Step 6. 发布复盘

- 记录阅读、在看、转发、评论
- 复盘后再更新知识库，闭环完成
