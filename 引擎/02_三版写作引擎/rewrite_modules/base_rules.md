# Rewrite 基础规则

## 核心原则

### 1. 结构保持（Structure Fidelity）
- ✅ 保留所有 h2 标题及其顺序
- ✅ 保留文章的逻辑层级（段落→小节→章节）
- ✅ 保留第一、第二、第三层标题数量（±1允许）

### 2. 内容完整性（Content Integrity）
- ✅ 保留所有关键事实、数据、引用
- ✅ 保留所有论据和论点映射关系
- ✅ 保留所有{{image:}}、{{link:}}、{{ref:}}锚点
- ❌ 不删除任何h2级别的主要观点

### 3. 文风转换（Tone Transformation）
- ✅ 根据平台和tone调整语气
- ✅ 调整用词密度和修辞手法
- ✅ 根据目标受众调整解释深度
- ❌ 改变核心观点或立场

### 4. 锚点保留（Anchor Preservation）
```markdown
原文: {{image: 纽约联储大楼外观}}
改写后: 保留在语义相关的段落中
规则: 同一类锚点不超过1个移位段距
```

## 映射规则

### h2 标题保留映射
```
初稿 h2      →  改写 h2
─────────────────────────
一、现状     →  一、现状
二、分析     →  二、分析
三、展望     →  三、展望
(顺序必须保持)
```

### 段落转换规则
```
原始段落(200字) 
  ↓ [根据平台/tone调整]
改写段落(180-220字)
  ↓ [检查锚点]
最终段落(包含所有锚点)
```

## 禁止操作

- ❌ 删除或合并h2标题
- ❌ 重新排列h2顺序
- ❌ 删除原文中的关键引用
- ❌ 改变论点的因果关系
- ❌ 丢弃{{image:}}、{{link:}}锚点
- ❌ 超过30%的字数变化（除非平台要求）

## 质量检查表

- [ ] h2顺序：完全保持
- [ ] 锚点完整：___个锚点全部保留
- [ ] 字数合规：改写___字（目标___-___字）
- [ ] 语气一致：与tone定义一致
