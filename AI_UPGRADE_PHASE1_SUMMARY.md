# 大圣媒体工作流 AI 升级总结

## 升级概览

**升级时间**：2026-04-04
**升级范围**：第一阶段核心环节（Brief + Rewrite）
**升级目标**：从硬编码规则系统转向 AI 驱动的创作流程

---

## 核心问题回顾

### 用户反馈
- "候选选题范围太窄"（只有 5 个）
- "题目特别硬，逻辑关系像生搬硬套"
- 要求"避免硬编码生成文章，利用 AI 结合 skills 流程规范"

### 根本原因
1. **过度过滤**：45,525 条原始记录 → 5 个最终选题（99.989% 过滤率）
2. **Brief 硬编码**：7 个固定模板，字符串拼接，无 AI 推理
3. **Rewrite 硬编码**：框架和策略文档存在但未实现，字数补充块固定为财经内容
4. **缺乏端到端 AI 集成**：各环节独立，没有统一的 AI 驱动流程

### 2. Rewrite 环节 - 完整应用框架和策略 ✅

**创建的新模块**：
- `scripts/framework_strategy_loader.py` - 框架和策略加载器
- `scripts/ai_content_expander.py` - AI 智能字数补充

**核心改进**：
- ✅ 完整应用 7 种写作框架（痛点型、故事型、清单型、对比型、热点解读型、纯观点型、复盘型）
- ✅ 完整应用 4 种内容增强策略（角度发现、密度强化、细节锚定、真实体感）
- ✅ 智能字数补充（根据选题类型和框架，不再是硬编码的财经块）
- ✅ 从 brief 继承框架和策略推荐

**框架应用流程**：
```
Brief 推荐框架和策略
    ↓
Draft 根据框架生成初稿
    ↓
Rewrite 应用框架结构 + 增强策略 + DNA 风格
    ↓
智能字数补充（根据框架类型）
```

**智能字数补充示例**：

旧系统（硬编码）：
```python
# 固定的财经内容块，不适配其他类型
补充块 = [
    "补充说明：判断是否进入'结构性改善'，必须同时满足三项条件...",
    "执行建议：把周度观察聚焦到变量变化...",
    "风险边界：若外部冲击超预期，需优先检查假设失效点..."
]
```

新系统（AI 驱动）：
```python
# 根据框架类型智能生成
if framework == "痛点型":
    补充 = "3 个常见错误做法 + 工具对比表 + 完整操作流程"
elif framework == "故事型":
    补充 = "关键对话 + 转折点细节 + 人物心理活动"
elif framework == "清单型":
    补充 = "隐藏推荐 + 避坑指南 + 使用场景对比"
# ... 其他框架类型
```

## 质量控制

### 自动检查
- Brief: 标题长度（15-30 字）、核心论点、证据需求、评分范围（0-10）
- Rewrite: 字数达标、风格匹配、平台适配

### 人工审核（保留 3 个 Gate）
- **Brief Gate**: 编辑从 10-15 个候选中选择 3-5 个进入 draft
- **Draft Gate**: 编辑确认事实准确性和逻辑
- **Rewrite Gate**: 编辑选择最佳版本并微调

### 回退机制
- 如果 AI 系统失败，自动降级到旧的规则系统
- 保留旧系统作为备份（`phase2_rebuilder.py`）
