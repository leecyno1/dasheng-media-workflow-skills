# Rewrite 约束条件（Constraints）

## 结构锚点约束

### h2标题锁定

```
禁止操作:
  ❌ 删除或添加h2标题
  ❌ 改变h2标题顺序
  ❌ 修改h2标题内容
  ❌ 将h2降级为h3

允许操作:
  ✅ 在h2下增加h3或h4（结构细化）
  ✅ 调整h2下的段落内容
  ✅ 合并h3（不涉及h2）
  ✅ 拆分段落（保持h2完整）

检查方式:
original_h2_count == rewritten_h2_count
original_h2_list == rewritten_h2_list (顺序)
```

### 锚点保留约束

```
锚点类型     保留规则                    移位容限
─────────────────────────────────────────────────
{{image:}}   必须保留，可移位           ±1段距
{{link:}}    必须保留，可移位           ±2段距  
{{ref:}}     必须保留，位置不变         0段距

示例:
原文段落3: {{image: 配图描述}}
改写段落2: 某某观点... {{image: 配图描述}}
改写段落4: 更多内容
→ ✅ 允许：移位1段（段3→段2或段4）

原文段落3: {{image: 配图描述}}
改写版本:  删除此锚点
→ ❌ 禁止：必须保留
```

## 内容完整性约束

### 事实与论据

```
约束项          规则                      验证方法
────────────────────────────────────────────────
数据准确性      所有数据必须与原文相同     逐一核对
论点因果        论点间的因果链不可改变     因果图对比
权威引用        原文引用必须保留          引用列表对比
案例完整性      原文案例不可删除          案例清单对比
```

### 观点保留

```
禁止:
  ❌ 删除原文的任何h2级别的核心观点
  ❌ 改变观点的逻辑顺序或因果关系
  ❌ 添加原文中完全没有的观点
  ❌ 删除支撑某观点的关键数据

允许:
  ✅ 补充新数据或案例（强化观点）
  ✅ 调整表述方式（改变语气但保持观点）
  ✅ 简化解释（降低复杂度但保留观点核心）
  ✅ 增加过渡句（改进逻辑流畅度）
```

## 字数约束

### 按平台和tone规范

```
平台     Tone    目标字数   允许范围    超出则
─────────────────────────────────────────────
WeChat   Hot     1300      1100-1500   需要重新平衡
WeChat   Normal  1000      800-1200    需要重新平衡
XHS      Hot     900       750-1050    需要重新平衡
XHS      Normal  650       500-800     需要重新平衡

超出容限的处理:
1. 删除重复内容（而非删除观点）
2. 合并相近段落（保留h2）
3. 简化案例说明（保留核心）
4. 改进文字效率（每句更精准）
```

## 语气约束

### Hot版本

```
必须:
  ✅ 开篇包含冲突、数据或反常信号
  ✅ 段落使用强动词（暴露、颠覆、潜伏）
  ✅ 包含具体论据（数据、案例）
  ✅ 结尾包含转发激励或问题激励

禁止:
  ❌ 使用"据悉""有报道"等软词汇
  ❌ 连续3个被动句
  ❌ 过度emoji（总数>6个）
  ❌ 绝对表述（必然、肯定、一定）
  ❌ 显得廉价（"听说""觉得""好像"）

验证:
强动词密度 ≥ 3个/500字
开篇冲突信号 = 1项
转发激励 ≥ 1项
```

### Normal版本

```
必须:
  ✅ 开篇清晰阐述问题和意义
  ✅ 段落使用SPA结构（Statement+Proof+Application）
  ✅ 包含权衡表述（一方面...另一方面）
  ✅ 结尾总结+隐含意义+前景思考

禁止:
  ❌ 感叹号
  ❌ 过度修辞或夸大
  ❌ "据说""好像"等模糊词汇
  ❌ "让我们来看"等老气表述
  ❌ "小编认为"等第一人称冗余
  ❌ 转发激励

验证:
SPA结构覆盖率 ≥ 80%
权衡表述数 ≥ 2处
论据充分度 ≥ 2个/段
```

## 平台特异性约束

### WeChat特异约束

```
配图:
  ✅ 保留所有{{image:}}锚点
  ✅ 锚点在段落间（不在段中）

排版:
  ✅ 使用h2作为主要分层
  ✅ 可使用分点（- 或 1. 2. 3.）
  ✅ 加粗关键词（≤3个/段）

互动:
  Hot: 必须包含"点在看"激励
  Normal: 可选评论问题

禁止:
  ❌ 使用过多hashtag（WeChat不推荐）
  ❌ emoji过多（>5个）
```

### XHS特异约束

```
标题:
  Hot: 20-35字，包含冲击词、数据或感叹
  Normal: 15-25字，直接切题
  必须: 保留核心概念可识别性

配图:
  Hot: ≥3张
  Normal: ≥2张
  位置: 段落间，每100字~1张图

排版:
  使用emoji加分点（Hot版本）
  段落控制在100-150字（XHS偏好短段）
  标签3-5个（Hot版本）

禁止:
  ❌ 段落超200字
  ❌ emoji滥用（≤6个）
  ❌ 传统学术用语
```

## 质量门槛

### 必须通过（HARD GATE）

```
1. 结构完整性
   h2数量和顺序100%保留 ✓
   所有{{image:}}/{{link:}}保留 ✓
   
2. 内容准确性
   所有数据与原文一致 ✓
   论点因果链不被破坏 ✓
   权威引用完整 ✓
   
3. 字数合规
   在目标范围内±10% ✓
   
4. 语气一致
   与版本定义相符（Hot/Normal） ✓
   不含禁用词汇 ✓
```

### 应该通过（SOFT GATE）

```
1. 可读性
   平均段落长度 150-200字（WeChat）
   平均段落长度 100-150字（XHS）
   句子平均长度 ≤20字
   
2. 论据充分
   每段≥2个论据（数据+案例）
   权衡表述≥2处（Normal版本）
   
3. 互动度
   结尾包含激励/问题（Hot版本）
   包含反思机制（Normal版本）
   
4. 视觉效果
   排版整洁，分层清晰
   emoji分布均匀（Hot版本）
   配图位置合理
```

## 冲突解决规则

当平台约束与字数约束冲突时：
```
优先级: 结构 > 内容完整 > 字数 > 语气微调

示例:
XHS需要900字(Hot)，但保留所有h2+论据需1050字
→ 选择: 保留结构和论据，接受字数超出至1050字
→ 而非: 删除论据以符合900字
```

当语气与结构冲突时：
```
优先级: 结构 > 语气

示例:
Normal版要求学术表述，但某观点需要加强表达
→ 选择: 保留强表述，标注需要编辑审核
→ 而非: 削弱表述以保持语气一致性
```
