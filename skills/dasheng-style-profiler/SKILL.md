# Dasheng Style Profiler

## 目标

将"像不像这个作者写的"从主观感觉，变成可执行、可复用、可迭代的显式资产，为 rewrite 阶段提供个人风格 DNA。

## 默认路径

- `DASHENG_WORKSPACE=${DASHENG_WORKSPACE:-/Volumes/PSSD/Projects/公众号文章}`
- 风格画像输出：`${DASHENG_WORKSPACE}/风格库/{作者}/风格画像.md`

## 输入要求

1. 作者名（必填）
2. 平台类型（公众号/小红书，必填）
3. 参考文章（至少 3 篇，推荐 5-10 篇）
4. 文章路径（可选，默认：`${DASHENG_WORKSPACE}/风格参考/{作者}/`）

## 工作流

1. 收集作者信息与样本路径
2. 读取参考文章，建立样本清单与统计信息
3. 按 14 维框架做深度分析（见 `references/style-14d-framework.md`）
4. 单独总结以下强制章节：
   - 标点符号偏好
   - 分块习惯
   - 段落配方
   - 叙述方法体系
   - 内容推进方式
5. 产出初版风格画像草稿并进入用户校准循环
6. 从参考文章提取 3-5 段 Few-shot 示例
7. 生成情绪曲线与触发词清单
8. 总结起承转合模板库
9. 输出最终风格画像到固定文件

## 用户校准规则

- 必须在初版画像后请求用户确认
- 用户指出偏差后必须定点修正，而非整体重写
- 至少完成 1 轮校准；用户明确"可用"后再固化最终版

## 14 维分析框架

使用下列四层共 14 维（详见 `references/style-14d-framework.md`）：

### 一、表层特征（4 维）
1. 用词习惯
2. 词汇与句式
3. 句式节奏
4. 语气与基调

### 二、结构特征（3 维）
5. 段落结构
6. 文章结构模式
7. 论证逻辑

### 三、深层特征（4 维）
8. 节奏感
9. 修辞手法
10. 情感表达
11. 思维特征

### 四、独特标记（3 维）
12. 专业度体现
13. 签名式标记
14. 起承转合微操

## 输出契约

按以下顺序输出：

1. `样本概况`
2. `14维风格分析`
3. `标点符号偏好`
4. `分块习惯`
5. `段落配方`
6. `叙述方法体系`
7. `内容推进方式`
8. `Few-shot 示例段落`
9. `情绪曲线分析`
10. `起承转合模板`
11. `可复用 Prompt Block`
12. `风格画像文件路径`

## 固化文件

最终画像默认保存到：
- `${DASHENG_WORKSPACE}/风格库/{作者}/风格画像.md`
- `${DASHENG_WORKSPACE}/风格库/{作者}/style_dna.yaml`（结构化数据）

如果用户未指定作者目录，降级保存到：
- `/tmp/{author_slug}-style-profile.md`

## 质量红线

- 不允许只做"词汇模仿"，必须覆盖结构、节奏、论证
- 不允许跳过标点符号偏好分析
- 不允许跳过分块习惯分析
- 不允许跳过段落配方和叙述方法体系
- 不允许跳过内容推进方式分析
- 不允许跳过用户校准直接定稿
- 不允许输出空泛结论，必须给示例、比例或频率
- 不允许生成与样本证据不一致的风格判断

## 与 rewrite 阶段集成

生成的风格画像可在 rewrite 阶段使用：

```bash
# rewrite 时指定个人风格 DNA
python3 scripts/rewrite_with_personal_dna.py --dna-path "${DASHENG_WORKSPACE}/风格库/{作者}/风格画像.md"
```

## 支持格式

- `.md`
- `.txt`
- `.docx`（先提取文本再分析）

## 交付模板

最终文档结构使用：
`references/style-profile-template.md`

## 与现有 DNA 的关系

- 现有 luxun/lemon 是预设 DNA，适用于通用场景
- 个人风格 DNA 是从用户历史文章提取的定制化风格
- rewrite 阶段可选择：预设 DNA 或 个人 DNA
- 建议：首次使用预设 DNA，积累 10+ 篇文章后提取个人 DNA
