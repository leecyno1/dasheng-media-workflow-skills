# 平台元数据生成规则

本文档定义 rewrite 阶段为各平台生成元数据的规则和约束。

---

## 元数据结构

每个改写变体在 `rewrite_manifest.json` 中包含 `platform_metadata` 字段：

```json
{
  "variant": "xhs_video_luxun_hot",
  "platform": "xiaohongshu",
  "platform_metadata": {
    "title": "AI+MA赛道Q1爆发：单季融资超百亿",
    "hashtags": ["#AI投资", "#财经", "#科技趋势", "#人工智能", "#投资机会"],
    "cover_text": "2026年Q1，AI+MA赛道迎来史无前例的融资热潮...",
    "description": "深度解析AI+MA赛道爆发背后的逻辑"
  }
}
```

## 平台规则

### 1. 小红书（Xiaohongshu）

**适用变体**：`xhs_video_luxun_hot`, `xhs_video_lemon_normal`

**元数据字段**：

| 字段 | 规则 | 生成方法 |
|------|------|---------|
| `title` | ≤20 字 | 从正文第一段提取核心论点，压缩为吸引眼球的标题 |
| `hashtags` | 5-8 个，格式 `#话题名` | 大话题×2 + 中话题×3 + 小话题×2-3 |
| `cover_text` | 前 300 字 | 直接截取正文前 300 字作为封面文案 |
| `description` | ≤50 字 | 一句话总结全文核心观点 |

**话题标签策略**：
- **大话题**（百万级）：`#财经` `#投资` `#AI` `#科技`
- **中话题**（10-100万）：`#AI投资` `#科技趋势` `#行业分析`
- **小话题**（精准）：从选题关键词提取，如 `#MA赛道` `#Q1融资`

**标题生成规则**：
- 必须包含核心数据点或反直觉论断
- 使用疑问句或感叹句增强吸引力
- 避免标题党，确保与内容一致

**示例**：
```json
{
  "title": "AI融资破百亿背后的真相",
  "hashtags": ["#AI投资", "#财经", "#科技趋势", "#人工智能", "#Q1融资"],
  "cover_text": "2026年Q1，AI+MA赛道迎来史无前例的融资热潮，单季融资额突破100亿美元。但这波热潮背后，隐藏着三个被忽视的结构性变化...",
  "description": "深度解析AI融资热背后的三大结构性变化"
}
```

### 2. 抖音（Douyin）

**适用变体**：从 `xhs_video` 变体衍生（使用相同视频文件）

**元数据字段**：

| 字段 | 规则 | 生成方法 |
|------|------|---------|
| `title` | ≤30 字 | 比小红书标题稍长，可包含更多信息 |
| `description` | ≤1000 字 | 完整的视频描述，包含关键论点和数据 |
| `captions` | 数组，每段 ≤15 字 | 从视频脚本提取口播文本，分段并标注时间轴 |
| `hashtags` | 1-3 个 | 选择最相关的 1-3 个话题 |

**字幕脚本格式**：
```json
{
  "captions": [
    {"time": "00:00", "text": "2026年Q1"},
    {"time": "00:02", "text": "AI融资破百亿"},
    {"time": "00:05", "text": "背后有三大变化"},
    {"time": "00:08", "text": "第一个是..."}
  ]
}
```

**字幕生成规则**：
- 每段 ≤15 字（中文字符）
- 时间轴间隔 2-5 秒
- 保持口语化，避免书面语
- 关键数据单独成段

**示例**：
```json
{
  "title": "AI融资破百亿！Q1三大变化解析",
  "description": "2026年第一季度，AI+MA赛道融资额突破100亿美元，创历史新高。本视频深度解析这波热潮背后的三大结构性变化：1. 从通用大模型转向垂直应用；2. 从技术驱动转向商业闭环；3. 从海外主导转向中美双引擎。",
  "hashtags": ["#AI投资", "#财经分析", "#科技趋势"],
  "captions": [
    {"time": "00:00", "text": "2026年Q1"},
    {"time": "00:02", "text": "AI融资破百亿"},
    {"time": "00:05", "text": "背后三大变化"}
  ]
}
```

### 3. B站（Bilibili）

**适用变体**：`bilibili_luxun_hot`, `bilibili_lemon_normal`（新增）

**元数据字段**：

| 字段 | 规则 | 生成方法 |
|------|------|---------|
| `title` | ≤80 字 | 完整标题，可包含副标题 |
| `description` | ≤2000 字 | 详细简介，包含时间轴、参考资料、数据来源 |
| `tags` | 最多 10 个，每个 ≤20 字 | 从内容提取关键词，覆盖主题、领域、类型 |
| `category` | 必选 | 科技、知识、财经等分区 |
| `cover` | 1146x717px (16:9) | 封面图路径 |

**标签策略**：
- **主题标签**（3-4个）：`AI投资` `科技趋势` `行业分析`
- **领域标签**（2-3个）：`人工智能` `风险投资` `商业分析`
- **类型标签**（2-3个）：`知识分享` `数据解读` `深度分析`

**分区选择规则**：
- 财经内容 → `财经` 分区
- 技术分析 → `科技` 分区
- 行业解读 → `知识` 分区
- 数据可视化 → `科技` 或 `知识` 分区

**简介格式**：
```
【视频简介】
一句话总结核心观点

【时间轴】
00:00 开场
01:30 第一部分
03:45 第二部分
...

【数据来源】
- 来源1
- 来源2

【参考资料】
- 资料1
- 资料2
```

**示例**：
```json
{
  "title": "AI融资破百亿！2026年Q1三大结构性变化深度解析【数据可视化】",
  "description": "【视频简介】\n2026年第一季度，AI+MA赛道融资额突破100亿美元，创历史新高。本视频通过数据可视化，深度解析这波热潮背后的三大结构性变化。\n\n【时间轴】\n00:00 开场：Q1融资数据概览\n01:30 变化一：从通用到垂直\n03:45 变化二：从技术到商业\n06:20 变化三：从海外到双引擎\n08:50 总结与展望\n\n【数据来源】\n- Crunchbase 全球融资数据库\n- PitchBook 风险投资报告\n- 36氪、IT桔子国内融资数据\n\n【参考资料】\n- 《2026 AI投资趋势报告》\n- 红杉资本《生成式AI市场地图》",
  "tags": ["AI投资", "科技趋势", "行业分析", "人工智能", "风险投资", "数据可视化", "知识分享", "深度分析", "商业分析", "Q1融资"],
  "category": "科技",
  "cover": "产物/04_Material/2026-04-05_075240_ai/topic-ai-ma-boom-q1-2026/images/cover.png"
```

### 4. 微信视频号（WeChat Channels）

**适用变体**：`wechat_channels_luxun_hot`, `wechat_channels_lemon_normal`（新增）

**元数据字段**：

| 字段 | 规则 | 生成方法 |
|------|------|---------|
| `title` | ≤30 字 | 简洁标题，突出核心观点 |
| `description` | ≤1000 字 | 视频描述，包含关键信息和引导关注 |
| `topics` | 1-3 个 | 微信话题，格式 `#话题名` |
| `location` | 可选 | 地理位置标签 |

**话题选择规则**：
- 优先选择微信热门话题
- 1-3 个，不超过 3 个
- 格式：`#话题名`（不带空格）

**描述格式**：
```
【核心观点】
一句话总结

【详细内容】
2-3 段文字说明

【关注我们】
获取更多财经分析
```

**示例**：
```json
{
  "title": "AI融资破百亿背后的三大变化",
  "description": "【核心观点】\n2026年Q1，AI+MA赛道融资额突破100亿美元，背后隐藏三大结构性变化。\n\n【详细内容】\n第一，从通用大模型转向垂直应用，资金开始追逐有明确商业场景的AI产品。\n\n第二，从技术驱动转向商业闭环，投资人更关注营收和用户增长。\n\n第三，从海外主导转向中美双引擎，中国AI创业公司融资占比首次超过30%。\n\n【关注我们】\n获取更多财经深度分析",
  "topics": ["#AI投资", "#科技趋势", "#财经分析"]
}
```

## 元数据生成流程

### 输入
- 改写后的完整文本（markdown 格式）
- 选题元数据（topic_id, keywords, framework）
- 目标平台（xiaohongshu, douyin, bilibili, wechat_channels）

### 处理步骤

1. **提取核心信息**
   - 识别核心论点（通常在第一段或结论段）
   - 提取关键数据点（数字、百分比、对比）
   - 识别反直觉论断或独特角度

2. **生成标题**
   - 根据平台字数限制压缩核心论点
   - 小红书：≤20 字，强调吸引力
   - 抖音：≤30 字，可包含更多信息
   - B站：≤80 字，可加副标题
   - 视频号：≤30 字，简洁直接

3. **生成话题标签**
   - 从选题 keywords 提取
   - 根据平台特点调整数量和格式
   - 小红书：5-8 个，分层级
   - 抖音：1-3 个，最相关
   - 视频号：1-3 个，微信热门

4. **生成描述文本**
   - 小红书：前 300 字作为封面文案
   - 抖音：完整描述，≤1000 字
   - B站：详细简介，包含时间轴和来源
   - 视频号：结构化描述，包含引导关注

5. **生成字幕脚本**（仅抖音）
   - 从视频脚本提取口播文本
   - 分段，每段 ≤15 字
   - 标注时间轴（间隔 2-5 秒）

6. **选择分区**（仅 B站）
   - 根据内容主题自动选择
   - 财经 / 科技 / 知识

### 输出
- 更新后的 `rewrite_manifest.json`，包含所有变体的 `platform_metadata`

---

## 质量检查

生成元数据后，必须进行以下检查：

### 字数检查
- [ ] 小红书标题 ≤20 字
- [ ] 抖音标题 ≤30 字
- [ ] B站标题 ≤80 字
- [ ] 视频号标题 ≤30 字
- [ ] 抖音字幕每段 ≤15 字
- [ ] B站标签每个 ≤20 字

### 格式检查
- [ ] 话题标签格式正确（`#话题名`）
- [ ] 小红书话题数量 5-8 个
- [ ] 抖音话题数量 1-3 个
- [ ] B站标签数量 ≤10 个
- [ ] 视频号话题数量 1-3 个

### 内容检查
- [ ] 标题与正文内容一致
- [ ] 无标题党或夸大
- [ ] 话题标签相关性强
- [ ] 描述文本完整清晰
- [ ] 字幕脚本口语化

---

## 错误处理

如果元数据生成失败或不符合规则：

1. **字数超限**：自动截断并添加省略号
2. **话题标签不足**：从内容关键词补充
3. **话题标签过多**：保留最相关的前 N 个
4. **分区未选择**：默认使用 `知识` 分区
5. **字幕时间轴缺失**：按 3 秒间隔自动生成

所有错误和自动修正必须记录到 `rewrite_manifest.json` 的 `warnings` 字段。
