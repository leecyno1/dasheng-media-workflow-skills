# 更新协议

## 什么时候必须更新本 Skill

- 用户改阶段顺序
- 用户改某阶段职责
- 用户改交付文件名或接口
- 用户把两个阶段合并或拆开
- 用户新增默认参数矩阵
- 用户指定新的长期默认工作方式

## 更新最小动作

每次确认改动后，至少同步更新：

1. `skills/dasheng-media-sop/SKILL.md`
2. `skills/dasheng-media-sop/references/*.md`
3. `引擎/03_全链路SOP工作流/STAGE_INTERFACES.md`

如果改动影响单阶段执行，再同步更新：

- `引擎/03_全链路SOP工作流/<对应阶段>_prompt.md`
- 对应单环节 skill（若该阶段已有 skill）

## 更新输出格式

每次更新后，对用户至少返回：

- 当前最新工作流顺序
- 本次改了哪些规则
- 哪些旧规则作废
- 后续默认按什么执行
- 如果改动影响脚本，还要返回新增/修订脚本路径

## 版本习惯

- 不强制单独维护 changelog
- 以当前文件内容为准
- 如需大改，可在文件头加一行更新时间
- 任何“临时命令”一旦连续使用两次以上，应优先固化为正式脚本并写入 skill
