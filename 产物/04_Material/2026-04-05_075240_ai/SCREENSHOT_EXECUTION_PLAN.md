## Execution Checklist

### Pre-Screenshot Setup
- [ ] Browser window set to 1920x1080 or larger
- [ ] Clear browser cache for clean screenshots
- [ ] Disable ad blockers if blocking content
- [ ] Use incognito/private mode to avoid paywalls (if needed)

### Screenshot Standards
- [ ] Minimum resolution: 1920x1080
- [ ] Format: PNG (lossless)
- [ ] Include source branding (logo, URL bar)
- [ ] Capture date/timestamp if visible
- [ ] Ensure text is readable at 100% zoom

### Quality Control
- [ ] Check resolution: `file screenshot.png | grep "1920 x 1080"` or higher
- [ ] Verify file size: >500KB (indicates good quality)
- [ ] Review for clarity: Text readable, no blur
- [ ] Confirm content: Key data points visible

### Post-Screenshot Documentation
For each screenshot, create quality report:
```markdown
# Screenshot Quality Report

**File**: screenshot_name.png
**Source URL**: [full URL]
**Captured Date**: 2026-04-05
**Resolution**: 1920x1080
**File Size**: XXX KB
**Content**: [Brief description]
**Key Data Points**: [List visible data]
**Usage**: [Where in article]
```

## Technical Notes

**Browser Recommendations**:
- Chrome/Edge: Best for consistent rendering
- Firefox: Good for privacy mode
- Safari: Use for Mac-specific sites

**Screenshot Tools**:
- macOS: Cmd+Shift+4 (select area) or Cmd+Shift+3 (full screen)
- Built-in browser tools: F12 > Device toolbar > Screenshot
- Third-party: Snagit, Lightshot (for annotations)

**File Naming Convention**:
`screenshot_[source]_[topic]_[date].png`

Example: `screenshot_cnbc_china_gdp_2026.png`
