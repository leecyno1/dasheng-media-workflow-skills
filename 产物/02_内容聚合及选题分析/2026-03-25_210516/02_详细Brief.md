## 题目一

### 母题
停火预期与战争冲击，正在如何重新定价油价、通胀与风险资产？

### 核心判断
- 这不是单纯的国际新闻，而是一个“地缘冲突溢价是否回吐”的资产定价问题。
- 现在最值得写的，不是复述战况，而是把“停火预期 -> 油价 -> 通胀预期 -> 资产波动”这条链讲清楚。

### 发散问题 1
停火如果只是短期窗口，油价这次回落还能持续多久？

### 初稿推理大纲
1. 开头先抛冲突：战场消息看似降温，但市场先交易的是停火预期而不是永久和平。
2. 解释本轮油价回落的核心变量：不是供需突然变化，而是地缘风险溢价被快速压缩。
3. 对比执行风险：停火方案已经出现，但是否会落地、会不会反复，是下一轮波动来源。
4. 延伸到资产：油价回落短期缓解通胀恐慌，但如果停火失败，市场会重新定价。
5. 结尾给读者观察框架：看停火执行、看油价波动中枢、看通胀预期是否同步回落。

### 发散问题 2
比起战况本身，再通胀预期是不是更值得成为主标题？

### 初稿推理大纲
1. 从市场语言切入：战争新闻本身已经泛滥，真正影响资产的是“通胀是否卷土重来”。
2. 用央行表态承接：如果企业因冲击更快提价，再通胀会重新成为政策和市场的主线。
3. 说明油价在这条链中的位置：它是风险事件里最敏感、最先被交易的传导器。
4. 讨论不同资产受影响方式：黄金、能源股、成长股、长债并不是同一套逻辑。
5. 结尾形成判断：如果要写投资讨论稿，主叙事应从“战争新闻”升级成“再通胀框架”。

### 发散问题 3
中国视角下，战火逼近中亚时，“陆上能源命脉”为什么突然变成可写切口？

### 初稿推理大纲
1. 先把视角从美以伊转回中国：国内读者更关心的是冲突如何影响中国能源安全。
2. 引出中亚与陆上能源通道：如果海运风险上升，陆上能源路径的重要性会被重新认识。
3. 把能源安全和价格波动联系起来：安全议题最终还是会反映到成本与定价上。
4. 延展到产业和政策：油气、管网、炼化、运输等都可能成为二级传播话题。
5. 收束成可写选题：这不是国际评论，而是一篇“中国如何对冲外部能源冲击”的框架稿。

## 题目二

### 母题
半导体与国产芯片，正在进入政策催化、产业分化和情绪过热并存的阶段。

### 核心判断
- 今天最密集的新鲜公众号素材集中在半导体链。
- 最值得写的不是单点新闻，而是“催化很多，但景气分化更大”的结构判断。

### 发散问题 1
深圳支持国产 GPU/NPU/CPU/DPU，这是一条短期政策消息，还是产业加速信号？

### 初稿推理大纲
1. 开头先亮明判断：政策表态本身不稀奇，关键是它和产业迭代节奏是否同步。
2. 解释为何 GPU/NPU/CPU/DPU 被同时点名：说明本轮支持不是单芯片，而是全链路基础设施。
3. 把政策与企业动态并读：中微新品、ST 涨价、设备和材料端信号同时出现，说明行业话题正在升温。
4. 但要写出分化：支持政策不等于所有环节都受益，落地效率和供需错配依然决定胜负。
5. 结尾给出编辑方向：可做“政策催化是否已进入产业验证期”的判断稿。

### 发散问题 2
中微新品、ST 涨价、ASML 罢工、芯片项目烂尾，能不能写成一篇“行业复苏里最危险的误判”？

### 初稿推理大纲
1. 用冲突开头：行业表面很热闹，但热闹不等于景气普涨。
2. 分三类信息拆开：设备端新品、海外供应链扰动、价格信号、项目烂尾风险。
3. 强调行业不是线性复苏，而是“头部受益、弱项暴露、尾部出清”并行。
4. 讨论投资者容易踩的坑：把所有催化都误读成全面景气，把个别科技新闻误读成产业确定性。
5. 结尾给结构化框架：哪些是可持续变量，哪些只是情绪脉冲。

### 发散问题 3
像“0.1nm 光刻机原型问世”这种极端科技消息，写作时应该怎么处理才不失真？

### 初稿推理大纲
1. 先承认它能带来流量，但不该直接当作产业结论。
2. 把消息源层级讲清楚：原型、实验室进展、量产能力、产业落地是完全不同的阶段。
3. 再讨论为什么这类消息容易被误用：因为它天然迎合“技术飞跃”的情绪叙事。
4. 给编辑一个处理原则：能写“市场在讨论什么”，不要直接写“行业已经怎样”。
5. 收束成方法论：半导体选题必须把新闻热度和产业验证拆开。

### 写作风险
- 不要把单条科技新闻直接升级成行业总趋势。
- 不要把政策支持和企业盈利改善画等号。
- 半导体题材很容易情绪化，标题可以强，但正文必须收得住。

## 题目三

### 母题
AI 工具与 OpenClaw 工作流，正在从“功能炫技”切换到“能不能接进生产流程”。

### 核心判断
- 这条线不是缺工具，而是缺筛选和落地框架。
- 今天的素材能支撑一篇“为什么工作流案例比工具大全更值得写”的稿子。

### 发散问题 1
为什么今年更值得写的不是“AI 新工具大全”，而是“工作流接入案例”？

### 初稿推理大纲
1. 开头点出疲劳感：工具越来越多，但多数读者已经不缺名单，缺的是落地路径。
2. 用今天的样本说明：Claude Code、OpenClaw、ima skills、Figma 自动出图，都是“接入流程”型而非单点工具型。
3. 解释传播逻辑：案例更容易建立信任，因为读者能立刻映射到自己的工作场景。
4. 引出写法升级：从介绍功能，转到讲“接在哪一步、替代什么、节省什么”。
5. 结尾给出方向：后续适合做连续栏目，而不是一次性盘点。

### 发散问题 2
为什么 OpenClaw、Claude Code、ima skills 这类内容适合做连续追踪栏目？

### 初稿推理大纲
1. 先解释它们的共同点：都不是单次体验型内容，而是会随着版本和接入方式持续迭代。
2. 再解释为什么适合自媒体：每次升级都能对应一个新的“实际用法”切片。
3. 引入用户价值：读者关心的不是参数，而是“这次更新我能不能立刻用上”。
4. 把栏目形态讲清：可以拆成接入教程、踩坑复盘、技能包盘点、真实工作流案例。
5. 收束成可执行建议：这条线适合做系列化，不适合只做一次热帖。

### 发散问题 3
从 Typeless 到 Figma 源文件，再到 OpenClaw 接微信，哪些题材值得继续追，哪些只是短期热闹？

### 初稿推理大纲
1. 先建立筛选标准：能否复现、能否稳定接入、能否带来真实生产动作。
2. 把不同工具放到这个框架里：设计类、代码类、自动化接入类的价值密度并不一样。
3. 强调“稳定性”比“首发炫酷”更重要：真正有持续价值的内容必须可被第二次、第三次复用。
4. 把今天的素材串起来：OpenClaw 接微信、skills 上线 ima，比单纯新功能更接近真实生产关系。
5. 结尾给初稿方向：做一篇“哪些 AI 技术热闹值得看，哪些工作流真正值得写”的判断稿。

### 写作风险
- 不能把技术圈热度直接等同于大众读者需求。
- 不能只写“新功能”，要写“接入动作”和“结果变化”。
- 若没有真实实测，不要虚构效率提升幅度。
